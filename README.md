Phase 1: Requirement and User Story

เอกสารขอบเขตงาน (Terms of Reference: TOR)
โครงการพัฒนาเว็บแอปพลิเคชันทำข้อสอบออนไลน์ (quizWebApp)
1. รายชื่อสมาชิกกลุ่ม
นางสาวปภัสสร ว่องธนากิจ
นางสาวศรุชา วิริยะจิตต์
นางสาวภัทรธิดา จารุจิตจำเริญ

2. ที่มาของปัญหาและความสำคัญของโครงการ
ในปัจจุบัน การเรียนการสอนออนไลน์มีการใช้งานระบบทำข้อสอบออนไลน์อย่างแพร่หลาย
อย่างไรก็ตาม ระบบทำข้อสอบจำนวนมากยังขาดการแสดงผลการวิเคราะห์คะแนนและสถิติที่ชัดเจน เช่น คะแนนเฉลี่ย คะแนนสูงสุด และคะแนนต่ำสุด ส่งผลให้ผู้เรียนไม่สามารถประเมินผลการเรียนรู้ของตนเองได้อย่างมีประสิทธิภาพ

ดังนั้น โครงการ quizWebApp จึงมีวัตถุประสงค์เพื่อพัฒนาเว็บแอปพลิเคชันสำหรับทำข้อสอบออนไลน์ ที่สามารถตรวจคำตอบ คำนวณคะแนน และวิเคราะห์ผลคะแนนในรูปแบบสถิติได้โดยอัตโนมัติ เพื่อช่วยเพิ่มประสิทธิภาพในการเรียนรู้และลดภาระของผู้สอนในการตรวจข้อสอบ

3. จุดประสงค์ของโครงการและประโยชน์ที่คาดว่าจะได้รับ
3.1 จุดประสงค์ของโครงการ
เพื่อพัฒนาเว็บแอปพลิเคชันทำข้อสอบออนไลน์โดยใช้ Node.js
เพื่อสร้างระบบตรวจคำตอบและคำนวณคะแนนอัตโนมัติ
เพื่อแสดงผลสถิติของคะแนนสอบ เช่น ค่าเฉลี่ย คะแนนสูงสุด และคะแนนต่ำสุด

3.2 ประโยชน์ที่คาดว่าจะได้รับ
ผู้เรียนสามารถประเมินผลการเรียนรู้ของตนเองได้อย่างชัดเจน
ลดภาระในการตรวจข้อสอบของผู้สอน
ผู้พัฒนาได้ฝึกทักษะการทำงานเป็นทีม การพัฒนาเว็บแอปพลิเคชัน และการใช้งาน GitHub

4. โครงสร้างข้อมูล (Data Structure) ที่คาดว่าจะใช้
Question
ใช้เก็บข้อมูลคำถาม ตัวเลือกคำตอบ และคำตอบที่ถูกต้อง
Quiz
ใช้เก็บชุดข้อสอบที่ประกอบด้วยคำถามหลายข้อ
Result
ใช้เก็บข้อมูลผลลัพธ์การทำข้อสอบ เช่น คะแนน และสถิติของผู้ใช้
โครงสร้างข้อมูลเหล่านี้จะถูกนำมาใช้ในการตรวจคำตอบ การคำนวณคะแนน และการแสดงผลสถิติ

5. ขอบเขตของโครงการ (Scope)
5.1 สิ่งที่ดำเนินการ (In Scope)
ระบบทำข้อสอบแบบปรนัย (Multiple Choice)
ระบบตรวจคำตอบอัตโนมัติ
ระบบคำนวณและแสดงคะแนนสอบ
ระบบแสดงสถิติพื้นฐาน ได้แก่ Average, Max และ Min

5.2 สิ่งที่ไม่ดำเนินการ (Out of Scope)
ระบบชำระเงิน
ระบบสมัครสมาชิกขั้นสูง
ระบบสอบที่มีความซับซ้อน เช่น การจับเวลาแบบพิเศษ หรือการสอบแบบปรนัยร่วมกับอัตนัย

8. กระบวนการพัฒนา วิธีการ และเครื่องมือ
ใช้กระบวนการพัฒนาแบบ Agile เพื่อให้สามารถปรับปรุงและพัฒนาระบบได้อย่างต่อเนื่อง
ใช้ GitHub Repository สำหรับควบคุมเวอร์ชันและทำงานร่วมกันภายในทีม
พัฒนา Backend ด้วย Node.js

9. สรุปขั้นตอนการทำ Requirement
การจัดทำ Requirement ของโครงการเริ่มจากการสัมภาษณ์ผู้ใช้งานเป้าหมาย เพื่อทำความเข้าใจปัญหาและความต้องการของระบบ จากนั้นนำข้อมูลที่ได้มาวิเคราะห์และจัดทำเป็น Functional และ Non-Functional Requirements รวมถึงกำหนดขอบเขตของโครงการให้ชัดเจน

10. สรุปการประชุม Retrospective
หลังจากการดำเนินงานในแต่ละช่วง ทีมงานได้จัดการประชุม Retrospective เพื่อทบทวนการทำงานที่ผ่านมา โดยมีการสรุปดังนี้
